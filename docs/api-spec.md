# API Specification

This document details the RESTful endpoints exposed by the FastAPI backend of the Pro-Grade AI Assistant. These endpoints facilitate client communication, system health monitoring, and analytics aggregation.

---

## 1. Chat Orchestration

### `POST /chat`

**Description:** The main AI inference pipeline orchestrator. It processes the user query, orchestrates the translation, NLP entity recognition, intent classification, session management, and asynchronous logging.

**Request Body (JSON):**

| Field | Type | Required | Description | Default |
| :--- | :--- | :---: | :--- | :--- |
| `message` | `string` | Yes | The raw user input/query. | N/A |
| `user_id` | `string` | Yes | Unique identifier for session persistence and context. | N/A |
| `lang` | `string` | No | The user's preferred language code (e.g., "en", "ur"). | `"en"` |

**Example Request:**
```json
{
  "message": "I need help tracking my order ORD-12345",
  "user_id": "user_9982",
  "lang": "en"
}
```

**Response (JSON):**

| Field | Type | Description |
| :--- | :--- | :--- |
| `response` | `string` | The final text response generated by the AI (translated if needed). |
| `intent` | `string` | The classified intent of the user's message (e.g., `order_tracking`). |
| `sentiment` | `float` | The calculated sentiment polarity score of the message (-1.0 to 1.0). |
| `entities` | `object` | Extracted entities (locations, order_ids, names, products). |
| `is_discovery` | `boolean` | Flag indicating if the response was pulled via automated discovery from memory. |
| `confidence` | `float` | The confidence percentage of the intent classification (0 to 100). |
| `escalated` | `boolean` | Flag indicating if the conversation breached the negative sentiment threshold and was escalated. |
| `debug` | `object` | Internal metrics (`search_hits`, `top_score`) from the Lucene fuzzy search. |
| `session_context` | `object` | Context variables (`user_name`, `state`, `message_count`) managed by Redis. |

## 2. System Health

### `GET /health`

**Description:** Liveness and readiness probe for the backend container. It verifies connectivity to the underlying services (Lucene AI Engine and Redis Session Store).

**Response (JSON):**

| Field | Type | Description |
| :--- | :--- | :--- |
| `status` | `string` | General health status (`online` or `degraded`). |
| `session_storage` | `string` | Indicates if `redis` is active or if the system fell back to `in-memory-fallback`. |
| `lucene_up` | `boolean` | Connectivity status to the Java Lucene microservice. |
| `version` | `string` | The current API version (e.g., `3.2.0`). |

**Example Response:**
```json
{
  "status": "online",
  "session_storage": "redis",
  "lucene_up": true,
  "version": "3.2.0"
}
```

## 3. Analytics & Metrics

### `GET /analytics`

**Description:** Endpoint specifically tailored for the React frontend's analytics dashboard. It triggers the `AnalyticsEngine` to parse persisted Kafka logs and aggregate insights.

**Response (JSON):**

Returns an object containing performance summaries, including:
*   `intent_distribution`: Array of objects mapping intents to their frequency.
*   `accuracy_proxies`: Array comparing successful resolutions vs fallbacks over time.
*   `escalation_rates`: Rate of interactions transferred to human agents.
*   `conversation_health`: Sentiment trends and message count averages.

*(See frontend Recharts integration for visual representations of this data).*

---

### `GET /metrics`

**Description:** An endpoint serving a dual purpose. By default, via the `metrics_middleware`, it exposes time-series metrics in standard Prometheus format for scraping. The explicit route returns lightweight JSON statistics for basic monitoring.

**Response (JSON version):**

| Field | Type | Description |
| :--- | :--- | :--- |
| `request_count` | `integer` | Total number of HTTP requests processed since startup. |
| `active_sessions` | `integer` | Number of active chat sessions currently stored in Redis/Memory. |
| `uptime_seconds` | `integer` | Total seconds the application has been running. |
| `redis_enabled` | `boolean` | Confirms if the Redis connection is active. |

**Example Response:**
```json
{
  "request_count": 1450,
  "active_sessions": 42,
  "uptime_seconds": 86400,
  "redis_enabled": true
}
```
